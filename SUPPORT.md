## üêõ Bug Reports

**Found a bug?** Please report it!

### Before Reporting

1. **Search existing issues:** Your bug may already be reported
2. **Check documentation:** Ensure it's not expected behavior
3. **Try latest version:** Bug may already be fixed
4. **Gather info:** Error messages, logs, reproduction steps

### How to Report

**Create an issue:** [New Bug Report](https://github.com/OpenwaterHealth/openwater-commons/issues/new/choose)

**Include:**
- Clear, descriptive title
- Steps to reproduce
- Expected vs. actual behavior
- System information (OS, versions)
- Error messages or logs
- Screenshots (if applicable)

**Template:** Our bug report template will guide you

**Response time:** 24-48 hours for initial response

### Security Bugs

**DO NOT** create public issues for security vulnerabilities.

**Email:** [security@openwater.health](mailto:security@openwater.health)

See our [Security Policy](SECURITY.md) for details.

2. **Submit for review:** Create PR in openwater-governance
3. **Community feedback:** 2-week comment period
4. **TSC decision:** Technical Steering Committee votes

## ‚ùì Frequently Asked Questions

### General

**Q: Is Openwater free to use?**
A: Yes! All software is Apache 2.0 licensed, hardware is CERN-OHL-P. Free for academic, research, and commercial use.

**Q: Can I use Openwater for clinical applications?**
A: Not yet - our devices are investigational only. FDA clearance is in progress.

**Q: How can I contribute?**
A: Many ways! Code, documentation, hardware designs, testing, community support. See [CONTRIBUTING.md](CONTRIBUTING.md)

**Q: Do you offer training?**
A: Yes! Video tutorials, documentation, and office hours are free. Custom training available for enterprise.

### Technical

**Q: What programming languages does Openwater use?**
A: Primarily Python (software), C/C++ (firmware), JavaScript (web tools).

**Q: What CAD software is used for hardware?**
A: SolidWorks, Fusion 360, FreeCAD. We export STEP files for compatibility.

**Q: Can I integrate Openwater with my existing systems?**
A: Yes! We have APIs and follow standard medical device protocols (DICOM, HL7 FHIR).

### More FAQs

Full FAQ: [docs.openwater.health/faq/](https://docs.openwater.health/faq/)
